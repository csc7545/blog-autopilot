아웃라인의 H2/H3 구조를 기반으로 각 섹션의 본문을 **마크다운 서식**으로 작성해주세요.

## 구조

- 아웃라인의 각 H2는 하나의 섹션(heading)이 됩니다.
- 아웃라인의 각 H3는 해당 섹션의 subsection(subheading)이 됩니다.
- H3가 없는 H2는 subsections를 비워두세요.

## 본문 서식 규칙

각 content 필드에 마크다운을 사용하세요:

- **굵게**: 핵심 키워드나 중요 정보는 `**굵게**`로 강조
- **리스트**: 나열이 필요한 정보는 `- 항목` 또는 `1. 항목`으로 작성
- **줄바꿈**: 문단 구분은 빈 줄(`\n\n`)로 구분
- **인용**: 전문가 의견이나 팁은 `> 인용문` 형태로

## 본문 작성 요구사항

- 각 content는 최소 3문단, 150자 이상으로 충분히 작성
- 공감형 서론으로 시작 (독자의 고민이나 상황에 공감)
- 구체적인 정보와 예시 포함
- 중간에 독자에게 질문 삽입 ("혹시 ~해보신 적 있으세요?")
- 경험 기반 문장 포함 ("직접 ~해봤는데요")
- 리스트를 적극 활용하여 가독성 높이기

## 출력 형식

JSON으로 출력하세요:
```json
{
  "sections": [
    {
      "heading": "H2 제목",
      "content": "H2 전체에 대한 도입 본문 (마크다운)",
      "subsections": [
        {
          "subheading": "H3 소제목",
          "content": "H3에 대한 상세 본문 (마크다운)"
        }
      ]
    }
  ]
}
```
